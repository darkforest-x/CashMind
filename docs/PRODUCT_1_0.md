# CashMind 1.0 产品范围

## 定位

CashMind 1.0 的目标是先把“无感自动记账”的最小闭环跑通：

```text
交易发生
-> iPhone 快捷指令自动触发
-> VPS 接收并保存交易
-> 浏览器/PWA 查看、补记、编辑和统计
```

1.0 不追求完整金融 App，而是追求一个每天真实可用的自动记账系统。

## 核心卖点

- 无感采集: 用户尽量不打开 App，不手动输入金额。
- 轻量配置: 一台 VPS、两个 token、一套 iPhone 快捷指令。
- 私有部署: 数据存自己的服务器，不依赖第三方记账平台。
- 可修正: 自动导入后，可以在浏览器端编辑分类、备注和金额。
- 可扩展: Wallet、短信、邮件、OCR 文本都走同一个服务端账单模型。

## 1.0 必须包含

- 手机浏览器适配，支持添加到主屏幕作为 PWA 使用。
- VPS API 和 SQLite 数据存储。
- 快捷指令写入接口:
  - `POST /api/ingest/wallet`
  - `POST /api/ingest/text`
- 浏览器端账单管理接口:
  - 账单读取、创建、更新、删除
  - 预算读取、设置
  - AI 分类
- 密钥权限隔离:
  - 快捷指令密钥只允许写入。
  - App 访问密钥允许已授权浏览器读取、编辑、删除。
- 部署和运维文档。

## 1.0 明确不做

- 原生 iOS App 发布、Xcode 签名、TestFlight、App Store。
- 自动读取 iPhone 上所有 App 通知。
- 自动抓取微信、支付宝、银行 App 内部账单。
- 多用户、团队协作、权限分组。
- 复杂报表和会计级对账。

## iOS 自动化边界

iOS 浏览器和普通 App 不能随意读取 Wallet、通知、短信、邮件和其他 App 数据。1.0 的“无感”来自系统允许的自动化入口：

- Wallet/Apple Pay 交易触发器。
- 指定短信发件人或关键词触发器。
- 邮件触发器。
- 用户主动分享或 OCR 后转发文本。

如果某个银行或支付 App 没有被 iOS 快捷指令暴露出来，就不能保证全自动采集，只能通过短信、邮件、账单截图 OCR 或手动补记兜底。

## 验收标准

1. 用户在 iPhone 上刷卡后，快捷指令可以把金额和商户写入 VPS。
2. 用户打开浏览器/PWA，可以看到新交易。
3. 未授权浏览器无法读取账单。
4. 快捷指令密钥只能写入，不能读取账单。
5. VPS 重启后 PM2 可以恢复服务。
6. 文档足够让下一次部署、token 轮换、快捷指令重建不靠口头记忆。

## 1.1 建议方向

- 绑定域名并开启 HTTPS。
- 改进 PWA 图标和离线缓存策略。
- 做一键导入快捷指令模板。
- 增加 CSV/Excel 导入导出和月度报告。
- 增加更细的自动分类规则和商户规则库。
